Purpose & Goals

  • Purpose: Provide a Rust-based UniFFI library for the Polkadot/Substrate ecosystem so that native iOS, Android, and desktop apps can integrate with Polkadot without writing custom FFI or SCALE‐codec code.
  • Goals:
    • Expose essential wallet functionality (import existing wallet, derive accounts, query balances, build & sign transfers, submit extrinsics).
    • Make Polkadot integration as easy as adding a single package to any Swift/Kotlin project.
    • Maintain a single source of truth in Rust, ensuring consistent cryptography & SCALE encoding across all platforms.
    • Offer an upgrade path to on‐device light‐client support (via SmolDOT) in future releases.

Positioning vs. SubXT & SmolDOT

Library Purpose Key Differences
SubXT High‐level Rust SDK for Polkadot/Susbtrate that wraps JSON‐RPC calls & SCALE encoding.
  • Designed primarily for Rust backends and CLIs.
  • Requires a remote RPC endpoint (unless you enable light‐client feature).
  • Typed, ergonomic API inside Rust; not natively consumable by Swift/Kotlin without FFI glue.
SmolDOT Rust‐based, no‐std light client for Polkadot/Substrate; can sync headers, verify finality, and run in‐app.
  • Provides a fully trust‐minimized client within the app (no external node required).
  • Lower‐level API: you must handle SCALE decoding and extrinsic building manually (unless wrapped by SubXT).
  • Large dependency footprint (libp2p, consensus code) to compile for mobile targets.
PolkaBind Rust → UniFFI bindings exposing a minimal wallet‐focused Polkadot API for Swift, Kotlin, Python, etc.
  • Wraps SubXT (default) to expose import wallet, get balance, build & sign transfer, submit tx, subscribe events—all via UniFFI.
  • Optional future feature: compile with SmolDOT under the hood for on-device light client.
  • Delivers a single package per language—no additional FFI or bridging required by app developers.

Roadmap

Phase Timeline Objectives
Phase 0: Initialization Q2 2025
  • Set up GitHub organization (PolkaBind).
  • Create initial Rust crate with UniFFI UDL.
  • Publish v0.1.0: “import wallet” and “get balance” on RPC.
Phase 1: Core Wallet APIs Q3 2025
  • Implement build & sign transfer and submit extrinsic.
  • Expose subscribe to events (e.g. balances.Transfer) in UDL.
  • Publish Swift & Kotlin packages (v0.2.0).
  • Work in Progress – some methods may change.
Phase 2: SmolDOT Integration Q4 2025
  • Add a feature flag (--features light-client) to compile SubXT with SmolDOT backend.
  • Expose on-device finality verification methods (e.g. syncHeaders, verifyBlock).
  • Publish v0.3.0: “RPC + Light-Client” universal SDK.
Phase 3: Ecosystem Growth 2026
  • Documentation portal (on polkabind.dev/docs).
  • Community contributions: add support for staking, governance, parachains.
  • Release v1.0.0: Stable API, production-ready for multiple use cases.
